V tradičních wiki systémech je každá stránka reprezentována třemi způsoby: HTML kód, stránka jako výsledek zobrazení tohoto kódu prohlížečem a uživatelsky editovatelný zdrojový kód, ze kterého serverem vytváří HTML kód. Posledně jmenovaný formát, známý jako wikitext, je psán ve zjednodušeném značkovacím jazyku, jehož styl a syntaxe může být v jednotlivých implementacích odlišná. Důvodem k tomuto návrhu je, že HTML se svou velkou zásobou vnořených tagů je velmi komplikovaný pro rychlou editaci a odvádí pozornost od obsahu stránky. To, že uživatelé nemohou využít veškerou funkcionalitu, kterou HTML poskytuje, jako je JavaScript a CSS, se jeví spíše jako výhoda, protože je tím zajištěna konzistence vzhledu.
